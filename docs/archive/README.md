title: "Archived Docs"
---

These files are kept for institutional memory, not as current source-of-truth docs.

Archive criteria:

- superseded design reviews that still explain a wrong turn worth remembering
- probe-specific audits that are useful as case studies, not active operating docs
- tactical optimization notes that are no longer part of the main architecture surface
- retired product tracks preserved for research reference, such as `extract`

Current canonical docs stay in the `docs/` root or under `docs/architecture/`.
